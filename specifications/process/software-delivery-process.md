# 软件交付过程约定（起步）

## 摘要

约定本团队（或个人）在软件交付中遵循的**过程原则**。可执行细节以 Agent 技能与 SOP 工件为准；本文只固定「必须留下什么痕迹」。

## 适用范围

- 适用：功能开发、修复、发布与线上故障跟进
- 非目标：不替代 `agent-skills/skills/` 中的步骤正文；不定义某一云厂商操作手册

## 原则

1. **先工件后编码（可豁免但须声明）：** 有用户可见行为变更时，至少保留可验证的验收标准（`A-PRD` 或等价记录）。
2. **完成以验证为准：** 声称完成前须有命令或检查证据（`A-VERIFY` 或等价）。
3. **发布须可回滚：** 生产变更需要书面回滚路径（`A-RELEASE`）。
4. **故障先止血：** 线上事故优先缓解，再复盘（`A-INCIDENT`）。

## 与技能的关系

| 活动 | 技能入口 | 工件 |
|------|----------|------|
| 路由 / 全流程 | `using-sdlc-skills` / `running-sdlc-playbook` | 见 `docs/sop-artifacts.md` |
| 需求 / 设计 / 发布 / 运维 | 对应自有技能 | `A-PRD` … `A-INCIDENT` |
| 计划 / TDD / 审查 | Superpowers 插件 | `A-PLAN` / `A-VERIFY` |

## 检查清单

- [ ] 变更对应验收标准已知
- [ ] 验证证据已保留
- [ ] 若发布：回滚路径已写
- [ ] 若事故：复盘改进项有负责人
